### 2026-06-13

- `[id-exp]` **swiss3dt full 6-mode results (18 cells, single seed 2026)**.
  Test denorm RMSE (°C), per_entity LSTM, 50-trial HPO each, dropout tuned
  {0,0.1}, batch 32, 30 epochs:

  | dataset | none | embedding | onehot | random | sinusoidal | coordinates |
  |---|---|---|---|---|---|---|
  | 1990   | 1.723 | 1.289 | 1.119 | 1.139 | 1.116 | 1.155 |
  | 2010   | 1.642 | 1.368 | 1.201 | 1.222 | 1.224 | 1.255 |
  | zurich | 1.553 | 1.378 | 1.237 | 1.249 | 1.259 | 1.276 |

  **Verified genuine**: dim params artifact-checked — each random/sin cell
  searched 22 distinct dims (1..28); best-trial checkpoint input width ==
  1 + best_dim (e.g. 2010-random best_dim=25 → width 26). HPO val scores
  share the test ordering (none > embedding > transparent), so it is not
  a test-set artifact.

  **research-critic caveats (do NOT over-claim)**:
  - SINGLE SEED → no variance; emb 1.289 vs sin 1.116 (0.17°C) may or may
    not exceed noise. The transparent modes are within ~0.04°C of each
    other → NOT separable / NOT rankable at one seed.
  - CONFOUND: transparent modes enlarge enc_in (concat), so they carry
    more LSTM input params (random 54 983 vs embedding 51 191 vs none
    50 887) — "identifier type" is entangled with input capacity.
  - Same domain (3 swiss water-temp sets) → "observed on swiss-river",
    not "generalises".
  - Defensible claim: *all identifier modes beat none by 25-35%; embedding
    and the transparent modes are the strongest, with the transparent
    modes consistently a touch lower in RMSE — at a single seed.*
  - TODO to harden: multi-seed (>=3) + paired test; a capacity-matched
    control (e.g. none with widened enc_in / random-projected padding).

- `[id-exp]` **Result corollary**: dropout choice split 0.0 vs 0.1 roughly
  evenly across cells (tuning it was justified, not a dead knob).
